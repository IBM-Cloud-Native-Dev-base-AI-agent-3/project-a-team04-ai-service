1. 데이터 수집이 상대적으로 쉽다.

그래서 음… 이 방식으로 가볼까 하는데. 약간 이런식이라는 것 같아. 

```python
샘플 데이터가 없어도 가능한 이유:
이들은 사전에 수조 건의 데이터를 학습한 **'완성된 모델'**을 백엔드에 가지고 있습니다. 
사용자가 페이지에 접속하는 순간 다음과 같은 요소들을 실시간으로 체크합니다.

브라우저 핑거프린팅: 브라우저 엔진 버전, 설치된 폰트, 화면 해상도, 하드웨어 가속 여부 등이 
'일반적인 사람의 설정'인지 확인합니다. 봇(Headless Browser)은 종종 여기서 흔적을 남깁니다.

행동 패턴 (Telemetry): 마우스가 버튼으로 이동할 때의 가속도, 클릭 시의 압력(모바일), 
키보드 입력 사이의 미세한 시간차 등을 봅니다. 봇은 물리적인 한계가 없어서 너무 규칙적이거나 너무 빠릅니다.

네트워크 평판: 접속한 IP가 이전에 스팸을 보낸 기록이 있는지, VPN이나 프록시를 경유했는지 등을 확인합니다.

즉, 개별 서비스 운영자가 샘플 데이터를 준비할 필요는 없지만, 서비스 제공사(클라우드플레어 등)는 
이미 거대한 모델을 돌리고 있는 것입니다.
```

대략적으로 살펴보면 

1. 다 할 필요는 없고, 그냥 로봇이 아닙니다 같은 화면이 뜨면 마우스를 이동해서 로봇이 아닙니다 라는 곳으로 이동해서 클릭하기까지의 데이터를 먼저 수집하면 되지 않을까? 
2. 반대로 셀레니움을 통해서 그 작업을 하는 크롤링을 만들어서 데이터를 수집하고? 
3. 그래서 각각의 정답과 오답 데이터셋을 학습시키는거야.
4. 그리고 만약에 모델이 다 만들어졌다고 하면, 그걸 보정작업을 거쳐서 모델을 완성하고 
5. fastapi를 통해서 실시간으로 사람인지 봇인지 판별 여부를 로그를 찍어서 회원가입 성공했는지 여부를 DB랑 연동해서 예측율이 몇퍼센트 였는지 기록하고. 
6. 당연히 이 프로젝트는 컨테이너를 통해 배포되어야 하고… 

뭐 이런식으로 돌아가야 하지 않을까? 그래서 앞으로 먼저 할 일들을 정리. 

### 1. 데이터 수집을 위한 간단한 프로젝트 만들기

- 로봇이 아닙니다 같은 간단한 프론트 화면이 필요하겠고, 회원가입 버튼이 있는 화면도 필요하겠고, 성공하든 실패하든 리다이렉션이 될 페이지도 하나씩 필요하겠고
- 로봇이 아닙니다를 누르기 까지의 데이터를 수집해야 하니까 그게 실시간으로 DB랑 연동되어서 기록이 되는 그런 fastapi 서버 프로젝트가 필요할 것 같아.

## 2026.03.26 이미지 캡챠

![Image](https://github.com/user-attachments/assets/1c38d259-ee0e-46a0-836c-cb6fee74ccd3)


상대적으로 회원가입에 넣기에 더욱 간단하다. 

이미 이미지 캡챠를 뚫는 머신러닝 모델을 작업해본 적이 있어서 샘플 데이터 200개 정도 보유하고 있다.

[https://github.com/lepture/captcha](https://github.com/lepture/captcha)

[https://captcha.lepture.com/image/](https://captcha.lepture.com/image/) 

```python
from io import BytesIO
from captcha.image import ImageCaptcha

captcha = ImageCaptcha()
data: BytesIO = captcha.generate('ABCD')
```

![image.png](https://github.com/user-attachments/assets/1f30c5aa-20d8-4540-969c-19aec282deb5)


원래 처음 생각은 이렇다. 텍스트를 입력하면 이런식으로 text → image로 캡챠를 만들어주는 라이브러리가 있다.

1. 생성된 이미지를 회원가입란의 하단부에 넣는다. 이 캡챠로 사람인지, 봇인지 여부를 판단한다.
2. 사용자는 이미지 캡챠를 입력하고, 서버로 회원가입 요청을 보낸다.
3. 서버에서는 가지고 있던 정답값과 사용자가 입력한 값을 비교해서 회원가입 성공여부를 판단하고 결과값을 리턴한다.

그런데 생각해보니까 음…이게 ai가 맞나? 

그러니까, `이미지 캡챠를 뚫는 공격자의 입장`이면 ai가 맞다. 머신러닝. 

왜냐하면, 

1. 어떤 모델이 있고 지도 학습을 해서 정답 이미지를 데이터 라벨링 작업을 하고 
2. 그것을 기반으로 (캡챠 이미지, 정답)을 학습을 시켜서 이미지만 보고 정답을 맞히는 모델을 만드는 것이기 때문이다. 
3. 그러면 샘플 데이터를 넣어서 어떤 이미지가 있을때 실제로 그게 맞는지 정답률을 예측해볼 수도 있고 말야. 

이런 경우라면, 사실 200개의 데이터는 아무런 의미가 없을 정도로 적은 양이므로 

- 예시: **YOLO(You Only Look Once)**나 EfficientNet 같은 모델을 허깅페이스(Hugging Face) 등에서 다운로드합니다.
- 여기에 사용자님이 가진 샘플 데이터를 Fine-tuning하여 "특정 사물을 더 잘 찾아내는 우리만의 캡챠 엔진"을 만들었다고 하면 완벽한 AI 프로젝트 스토리가 됩니다.

이런 방식으로 파인튜닝을 시켜서 사용하는 방법도 생각했다. 

그런데, 음 결정적으로 설득력이 좀 떨어진달까… 

근본적으로, 

> 현재 사용하시려는 lepture/captcha 같은 라이브러리는 머신러닝 기반이 아니라, 전통적인 컴퓨터 그래픽스 알고리즘을 사용합니다. 즉, 특정 폰트를 불러와서 좌표를 비틀고(Warping), 랜덤하게 선(Noise)을 긋는 수학적 연산일 뿐 '학습'된 모델이 생성하는 게 아닙니다.
> 

이런 것도 좀 있고. 

검증하는 과정에서 지금 수업에서 하고 있는 사이킷런이나 파이토치 같은 것들의 사용도도 너무 적은 것 같고.
