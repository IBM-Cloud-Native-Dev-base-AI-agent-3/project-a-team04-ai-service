# project-a-team04-ai-service

https://github.com/lepture/captcha

Python의 `captcha` 라이브러리를 활용해 직접 시스템을 구축하고, 이를 머신러닝 학습 데이터로 연결하려는 아이디어는 **개발 실습과 데이터 사이언스 공부 측면에서 아주 훌륭한 프로젝트**입니다.

다만, 실제 서비스 적용 여부에 따라 고려해야 할 득과 실이 명확합니다. 기술적인 관점에서 제 생각을 정리해 드릴게요.

---

### 1. Python `captcha` 라이브러리 활용 (이미지 생성)
회원가입 로직에 넣는 것은 간단합니다. 서버(FastAPI 등)에서 이미지를 생성해 클라이언트에 보내고, 사용자가 입력한 값과 서버 세션에 저장된 값을 비교하면 됩니다.

* **장점:** 외부 API(구글 등)에 의존하지 않고 독립적으로 운영 가능하며, 디자인을 서비스 톤앤매너에 맞게 커스텀할 수 있습니다.
* **단점:** 서버 리소스를 사용하여 이미지를 생성해야 하므로 접속자가 많아지면 부담이 될 수 있습니다.

### 2. 데이터 샘플 학습 및 머신러닝 (OCR 모델 만들기)
직접 만든 캡차 이미지를 정답(Label)과 함께 저장하여 **'캡차를 깨는(Solving) 모델'**을 만드는 시도는 매우 추천합니다.

* **학습 과정:**
    1.  `captcha` 라이브러리로 수만 장의 이미지와 텍스트 쌍을 생성합니다.
    2.  CNN(Convolutional Neural Network)이나 RNN(LSTM/CTC) 계열의 모델을 사용해 이미지 속 글자를 맞히는 학습을 진행합니다.
    3.  이 과정을 통해 **"내 캡차가 얼마나 보안에 취약한가?"**를 역으로 테스트해 볼 수 있습니다.



### 3. 기술적 제언: "공부용 vs 실무용"

#### 🟢 공부용 (매우 추천)
직접 캡차를 생성하고, 그것을 다시 뚫는 AI 모델까지 만들어보는 것은 **전체적인 데이터 흐름(Data Pipeline)을 이해하는 데 최고**의 커리큘럼입니다.
* 데이터 생성 -> 전처리 -> 모델 설계 -> 학습 -> 평가의 사이클을 모두 경험할 수 있습니다.
* FastAPI와 연동하여 실시간으로 모델이 캡차를 얼마나 잘 맞히는지 대시보드를 만들어보는 것도 좋은 포트폴리오가 됩니다.

#### 🔴 실무용 (주의 필요)
실제 서비스의 보안이 목적이라면 직접 만든 숫자 캡차는 위험할 수 있습니다.
* **AI의 역설:** 본인이 학습시킨 모델이 99% 확률로 캡차를 맞힌다면, 공격자도 똑같이 모델을 만들어 회원가입을 뚫을 수 있다는 뜻입니다.
* **보안 취약성:** 고정된 폰트나 일정한 패턴의 노이즈는 최신 OCR 모델(EasyOCR, Tesseract 등)에 매우 쉽게 뚫립니다.
* **사용자 경험:** 사람은 숫자를 잘못 읽어 짜증을 느끼는데, 봇은 오히려 더 잘 통과하는 상황이 생길 수 있습니다.

### 💡 결론 및 제안
지금 진행 중인 **FastAPI와 Pandas 기반의 스터디 프로젝트**의 연장선으로 본다면, 이 시도는 **"완벽한 실습 주제"**입니다.

1.  **1단계:** Python으로 6자리 숫자 캡차 생성 및 검증 로직 구현.
2.  **2단계:** 생성된 이미지를 만 단위로 저장하여 학습 데이터셋 구축.
3.  **3단계:** PyTorch나 TensorFlow를 이용해 이 숫자를 판독하는 모델 생성.
4.  **4단계:** 직접 만든 모델로 내 웹사이트의 캡차를 공격해보고, 보안을 강화하기 위해 노이즈를 더 섞거나 굴곡을 더 주는 방식의 '적대적 학습' 시도.

이런 식으로 접근하시면 데이터 분석과 백엔드 개발 역량을 동시에 쌓는 데 큰 도움이 될 것입니다. 혹시 구체적인 모델 구조나 데이터 저장 방식(MongoDB 활용 등)에 대해 더 궁금한 점이 있으신가요?
